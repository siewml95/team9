Twitter API

This an the official API developed by Twitter.Inc. It allow developer to extract tweets from Twitter Database.

We will connect Twitter API to node.js to retrieve the tweets. We can take user inputs and concatenate them into a query to be send through Twitter API. Tweets retrieved will then be stored in SAP HANA Database.

It's free to use. The only publicly available API that allow developer to extract tweets. Work well with Node.js.

https://dev.twitter.com/rest/public

Node.js

Node.js is an interface to the V8 JavaScript runtime. It is mainly used for developing server-side web application. It is written mainly in JavaScript.

We will connect Twitter API to node.js to retrieve the tweets. We will use node.js to render a html with neccessary input such a text input for user to insert keyword to be searched. We will also use the Node.Js to render the dashboard shown in the mock up dashboard.

One of our member has some experience in Node.js. Node.js is very simple to be used and deployed. There are twitter api module for node.js.

https://node,js.org

SAP HANA

SAP HANA is a computing platform that focuses on relational database. It combines database,application processing and integration services on a single platform.

Tweets Extracted from Twitter API will be inserted into the database handled by SAP HANA. Tweets in the database will be processed by SAP HANA's algorithms into sentiment data. SAP HANA will also process the data to find out which users are more influential. The processed tweets will be stored in another table which we connect to the dashboard.

It has a free trial version which is sufficient to be used. It is easy to use and there are complete tutorials on how to use this platform.

go.sap.com

Historical Data

Pros

More more efficient (easier to collect a large amount of data as the data is already there compared to real time where you have to wait a certain period of time to sufficient collect data and so its less time consuming). More consistent (usually product cycle demands reports are in daily, weekly, monthly or annually and its not possible with real time). Extract more data than realtime data in the same amount of time.

Cons

Softwares that allow you to extract tweets are usually realtime (this will reduce the number of available software to use) [as long as we are able to find a good software that provides historic then we can use it]. We have to pay to get the historical tweets.

RealTime Data

Pros

Tweets are more up to date. Extracting realtime tweets is free.

Cons

Takes more time to get the tweets (this may be negligible depending on the circumstances). The speed depends on the popularity of the products (for more famous products the data will be more as more tweets will be created per second).

We have decided to use Realtime Data.
